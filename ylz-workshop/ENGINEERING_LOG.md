### Workflow Decisions

**On quote acceptance — auto-trigger:**
1. Create job on job board (Kanban)
2. Create MRPeasy manufacturing order
3. Email job sheet to workshop

**Job starting stage:** Requires Engineering (engineer fills in eng fields before fab starts)

**Job sheet generation:** Engineer reviews first — NOT auto on quote acceptance

**Truck + Trailer combo:** Two separate job sheets

**Engineering fields (mainrunner width, hoist box height, pivot):** Stay on job sheet, NOT in configurator. Configurator is for sales-facing fields only.

**Dimensions at quote time:** Sometimes known, sometimes not. System must support "TBD" for dimension fields at quote stage.

**Truck brand pricing:** Brand does NOT affect price — only dimensions differ.

**Drawing number:** Field on job, populated after job creation. Not generated by system.

**Sign-offs:** Yes — each production section (Fab, Paint, Fitout etc.) needs sign-off field on job sheet.

**Combo job sheets:** Two separate sheets (one truck, one trailer).

**Handover sheet:** Prepared in advance. Contains customer details, full spec, warranty info.

**VIN/compliance timing:** During fabrication, when serial number is assigned.

## Key Decisions Made

### D001: Standalone HTML Tool Pattern — CONTINUE
The quote-builder.html pattern works well. Fast to iterate, no build step, proven.
Decision: Extend existing quote-builder.html for the configurator. Port to React later if needed.

### D002: Cache-First MRPeasy
MRPeasy integration must be cache-first. Quoting must work without live API calls.
Decision: Implement cache tables, manual sync button, staleness warnings.

### D003: Build Record as Single Source of Truth
A single build record (quote config JSON) drives ALL outputs.
Decision: Extend existing Quote.configuration JSON field with full JSC field set.

### D004: No Greenfield Rebuild
Extend the existing system. Do not rebuild what already works.

### D005: Engineering Fields NOT in Configurator
Confirmed by Nathan: mainrunner width, hoist box height, C/L pivot — these stay on the job sheet.
The configurator captures sales-facing fields only.

### D006: Suppliers are Binnotto / Camelliri / SAF / SSAB
NOT Hyva/Palfinger, NOT Razor/EziTarp/Pulltarp, NOT BPW/Fuwa.
Existing JSC dropdowns need these added as primary options.
Old options retained for backward compatibility with existing jobs.

### D007: MRPeasy Import Format
Liz imports a form. System must generate this form in Liz's required import format.
Need to confirm exact format from Liz before building the export.

### D008: Monday.com Is Temporary
Build the system to eventually replace Monday entirely.
Monday integration is a bridge, not a dependency.

### D009: Two Job Sheets for Combos
Truck + trailer builds always generate two separate job sheets.

### D010: VIN Generator — Link Don't Duplicate
VIN plate generator is already built. New system links to it from the job record.
